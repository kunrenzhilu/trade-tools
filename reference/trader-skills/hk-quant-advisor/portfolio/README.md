# Portfolio 运行时数据目录

此目录用于存储 AI 运行时产生的持久化数据文件。

**注意**：此目录下的运行时数据文件（`.json`）已被 `.gitignore` 排除，不纳入版本管理。

## 初始化模板

`.templates/` 子目录包含全部 11 个运行时 JSON 文件的**空白初始化模板**。首次启动时，系统会将模板复制到 `portfolio/` 根目录并执行账户信息采集流程。若运行时文件缺失或损坏，也可从此目录恢复空白结构。

> 模板文件纳入版本管理，确保首次启动不依赖 File_Skill 的自动创建行为。

## 运行时文件说明

| 文件名 | 用途 | 更新时机 |
|--------|------|----------|
| `account_state.json` | 账户总资金、可用现金、总市值 | 每次成交回报后、每日收盘后 |
| `positions.json` | 所有持仓详情（代码、数量、成本价、当前价、浮盈亏、持仓天数、最高价） | 每次成交回报后、盘中每30分钟刷新 |
| `trade_log.json` | 所有已完成交易记录（含未执行标记） | 每次成交回报后 |
| `pending_orders.json` | 已发出但未收到回报的指令 | 指令发出时写入，收到回报后移除 |
| `strategy_params.json` | 运行时策略参数（基于默认参数 + 热更新） | 参数热更新时 |
| `blacklist.json` | 黑名单标的列表及原因 | 触发时实时更新 |
| `error_patterns.json` | 错误模式库（8类错误模式及纠正措施） | 每日收盘后检测更新 |
| `optimization_log.json` | 参数优化建议历史及采纳记录 | 月度报告后写入 |
| `signal_snapshots.json` | 每个交易信号触发时的完整条件快照 | 每次信号生成时写入 |
| `pending_notifications.json` | 通讯渠道发送失败时缓存的待补发指令单 | 发送失败时写入，成功补发后清除 |
| `stock_pool.json` | 核心池与观察池标的列表及统计信息 | 每周全量刷新、每日增量刷新 |
