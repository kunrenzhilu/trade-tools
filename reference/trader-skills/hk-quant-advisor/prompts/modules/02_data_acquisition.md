## 第二章：多源数据获取与分析维度

> 所有外部数据访问均通过 OpenClaw 标准化 Skill 接口执行。数据源完整配置详见 `config/data_sources.json`。
> **核心原则**：数据是一切决策的基础，宁可因数据不足而不操作，也不可基于失真数据做决策。

---

### 2.1 数据源 Skill 架构

> **设计理念**：hk-quant-advisor 本身不负责底层数据获取（不构造 URL、不解析网页、不管理 API 连接），所有数据获取通过调用外部数据源 Skill 完成。系统只关注"调用 Skill 获取数据 → 分析决策 → 输出指令"。

#### 2.1.1 可用数据源 Skill

**免费数据源（无需 API Token，必选安装）：**

| Skill 名称 | 数据源 | 特点 |

| `tecent-finance` | 腾讯证券（外网） | 外网 ClawHub 渠道，港股实时行情、历史K线，与 westock-data 同源 |

#### 2.1.3 降级与切换规则

1. **逐级降级**：当前 Skill 请求失败（连续 3 次）→ 自动切换到优先级链中下一个 Skill
2. **交叉验证**：关键决策数据（触发买入/卖出）须至少调用 2 个不同 Skill 获取数据并交叉确认
3. **全链失败**：若某数据维度所有 Skill 均不可用 → 该维度标记为"不可用"，系统根据数据完整度判定规则决定后续行为
4. **恢复回切**：失败 Skill 每 5 分钟自动探测恢复，连续 3 次成功后回切为主 Skill
5. **抗震荡机制**：若同一数据源在 60 分钟内发生 ≥ 2 次「降级→恢复→再降级」循环，则后续恢复回切的成功次数要求从 3 次递增为 6 次（指数退避，倍率=2），冷却期为 30 分钟。具体参数见 `data_sources.json` 的 `degradation_rules.anti_flap_*` 配置

#### 2.1.4 不由数据源 Skill 覆盖的数据获取

| 数据项 | 获取方式 | 说明 |

| PDF 财报解析 | `File_Skill (PDF Parser)` | 本地解析能力 |
| 新闻情感分析 | `NLP_Skill` | 本地 NLP 能力 |

系统决策依赖以下五个维度的数据，缺少任何一个核心维度（行情/基本面）时不生成新开仓信号：

| 维度 | 重要性 | 数据内容 | 刷新频率 |
|------|--------|----------|----------|
| ① 行情数据 | **必需** | 实时/延迟报价、K线、成交量、买卖盘 | 盘中每30分钟 |
| ② 基本面数据 | **必需** | 财报、估值、审计意见、十维评分卡 | 每日盘后/公告触发 |
| ③ 资金流向 | 重要 | 港股通资金、CCASS券商持仓、卖空数据 | 每日盘后18:00 |
| ④ 舆情情绪 | 辅助 | 新闻情感、社交热度、VHSI恐慌指数 | 盘中/盘后 |
| ⑤ 宏观环境 | 辅助 | 汇率、行业PMI、美联储利率 | 每日/事件触发 |

> **数据完整度判定**：行情+基本面可用 → 正常运行；仅行情可用 → 仅执行止损/止盈监控；全部不可用 → 进入休眠模式。
> 
> **"基本面可用"定义**：指最近一次盘后刷新获取的基本面数据存在且未过期（最近 7 个交易日内有更新），而非要求盘中实时获取基本面数据。

#### 2.1.5 股票代码标准化

所有外部数据源 Skill 返回的股票代码，在进入系统处理流程前**必须先执行标准化**：

| 原始格式 | 标准化后 | 说明 |
|----------|----------|------|
| `700.HK` | `00700.HK` | 补零至 5 位 |
| `0700.HK` | `00700.HK` | 补零至 5 位 |
| `00700.HK` | `00700.HK` | 已是标准格式，无需处理 |
| `700` / `0700` | `00700.HK` | 补零 + 追加 `.HK` 后缀 |

**标准化规则**：
1. 提取代码中的纯数字部分，左侧补零至 **5 位**
2. 追加 `.HK` 后缀，统一为 `XXXXX.HK` 格式
3. 标准化后再进行后续的 Schema 校验、技术指标计算、交叉验证等操作
4. 若代码中包含非数字字符（除 `.HK` 后缀外），标记为"代码格式异常"并跳过该条数据

### 2.2 异常数据检测规则

所有数据在使用前必须通过以下异常检测：

#### 价格异常
- **跳变检测**：单笔成交价偏离前一笔超过 **20%**（非公告日），标记为可疑数据
- **处理方式**：可疑数据不纳入技术指标计算，等待后续 3 笔成交确认趋势后再更新
- **公告日豁免**：若当日有业绩公告或重大事项公告，放宽跳变阈值至 **30%**

#### 成交量异常
- **突增检测**：单日成交量 > MA(V,20) × 10 倍，标记为"异常放量"
- **处理方式**：交叉验证至少 2 个数据源 Skill，确认非数据错误后方可采信
- **合理放量**：VR 在 2.0-8.0 之间且有明确催化因素（公告/事件）视为合理放量

#### 时间戳验证
- 行情时间戳偏差 > **30 秒** → 标记"延迟"，不触发新开仓信号
- **超时处理**：偏差 > 60 秒视为数据源 Skill 不可用，触发自动切换到下一优先级 Skill

#### 交叉验证
- 关键决策数据（触发买入/卖出的数据点）须至少 2 个独立数据源 Skill 确认
- 验证项目：最新价误差 < 0.5%、成交量误差 < 20%、涨跌方向一致

### 2.3 技术指标本地计算引擎（两种模式共享）

以下指标基于抓取到的历史 K 线数据，由 OpenClaw 本地计算引擎生成，确保算法一致性和可复现性：

#### 均线系统
- **简单移动平均线 MA**：`MA(N) = SUM(CLOSE, N) / N`，周期：5/10/20/30/60/120
- **指数移动平均线 EMA**：`EMA(N) = CLOSE × 2/(N+1) + EMA_prev × (N-1)/(N+1)`，周期：20/60
- **用途**：MA 用于日线趋势判断和支撑压力位；EMA 用于周线大趋势确认

#### MACD（12,26,9）
```
DIFF = EMA(CLOSE, 12) - EMA(CLOSE, 26)
DEA  = EMA(DIFF, 9)
MACD柱 = (DIFF - DEA) × 2
```
- **信号**：DIFF 上穿 DEA = 金叉（做多信号）；DIFF 下穿 DEA = 死叉（做空信号）
- **背离**：价格创新高但 MACD 柱未创新高 = 顶背离（卖出警告）

#### KDJ（9,3,3）
```
RSV = (CLOSE - LOW9) / (HIGH9 - LOW9) × 100
K = SMA(RSV, 3, 1)    // K = 2/3 × K_prev + 1/3 × RSV
D = SMA(K, 3, 1)      // D = 2/3 × D_prev + 1/3 × K
J = 3K - 2D
```

> **SMA 函数消歧义说明**：此处 `SMA(X, N, M)` 为**加权递推平均**（Weighted Recursive Moving Average），公式为 `Y = (M × X + (N-M) × Y_prev) / N`，**并非简单移动平均**（Simple Moving Average）。当 N=3, M=1 时等价于 `Y = 1/3 × X + 2/3 × Y_prev`。这是中国证券行业通用的 KDJ 计算方式（同花顺、通达信等均采用此算法）。

- **超买超卖**：J > 105 超买区；J < -5 超卖区
- **金叉/死叉**：K 上穿 D = 金叉；K 下穿 D = 死叉

#### RSI（14）
```
RS = SMA(MAX(CLOSE-CLOSE_prev, 0), 14) / SMA(ABS(MIN(CLOSE-CLOSE_prev, 0)), 14)
RSI = 100 - 100 / (1 + RS)
```
- **区间**：RSI < 20 超卖（策略四关注区）；RSI 35-45（策略二关注区）；RSI 55-80（策略一关注区）；RSI > 80 超买

#### 布林带 BOLL（20,2）
```
中轨 = MA(CLOSE, 20)
上轨 = 中轨 + 2 × STD(CLOSE, 20)
下轨 = 中轨 - 2 × STD(CLOSE, 20)
```
- **用途**：价格触及下轨 + 缩口 = 潜在反转信号；价格突破上轨 + 扩口 = 趋势加速

#### ATR（14）
```
TR = MAX(HIGH-LOW, ABS(HIGH-CLOSE_prev), ABS(LOW-CLOSE_prev))
ATR = SMA(TR, 14)
```
- **用途**：动态止损计算（止损距离 = N × ATR）；仓位计算（单笔风险金额 / ATR = 股数）

#### VWAP（日内）
```
VWAP = Σ(典型价格 × 成交量) / Σ(成交量)
典型价格 = (HIGH + LOW + CLOSE) / 3
```
- **用途**：盘中参考价，高于 VWAP 买入偏强势，低于 VWAP 卖出偏弱势
- **近似计算**：基于分钟 K 线数据近似计算

#### OBV（能量潮）
```
IF CLOSE > CLOSE_prev THEN OBV = OBV_prev + V
IF CLOSE < CLOSE_prev THEN OBV = OBV_prev - V
IF CLOSE = CLOSE_prev THEN OBV = OBV_prev
```
- **用途**：OBV 创新高而价格未创新高 = 量价背离（潜在突破信号）

### 2.4 资金流向与持仓数据

**来源与技能**：
- **港股通资金**：每日 18:00 后，通过数据源 Skill 获取**每日港股通十大成交活跃股**及**整体资金流向数据**（优先使用 `tushare-finance` 的港股通资金接口，降级到 `akshare-finance`）。同时调用 `Browser_Skill` 抓取港交所"互联互通"栏目数据进行交叉验证。统计 5/10/20 日累计净流入。
- **CCASS 券商持仓**：每个交易日 20:00 后，调用 `Browser_Skill` 访问港交所 CCASS 系统，下载最新持股数据，计算头部券商席位（摩根大通、汇丰、花旗、高盛等）持股日变化率和周变化率。
- **卖空数据**：每日 18:30 后，调用 `Browser_Skill` 抓取港交所卖空记录汇总。
- **个股资金流向**：盘中通过数据源 Skill 获取实时资金流向数据（优先 `akshare-finance`，降级到 `westock-data`）。

**分析规则**：

| 信号 | 判定条件 | 含义 |
|------|----------|------|
| 多头共振 | 南向资金近 5 日净流入 **且** 外资行持股比例周环比增加 > 0.5% | 内外资金同向看多 |
| 筹码集中 | 前十大券商席位持股比例周环比上升 | 主力吸筹信号 |
| 南向大举介入 | 单日南向净买入占该股日成交额 > 30% | 高度关注信号 |
| 卖空承压 | 卖空比率（当日卖空额/总成交额）> 15% | 短期下行压力 |

### 2.5 市场情绪与新闻舆情

**来源与技能**：
- **新闻**：通过数据源 Skill 获取港股相关新闻快讯（优先 `tushare-finance`，降级到 `akshare-finance`）。对获取的新闻按个股名、行业、政策关键词过滤。若数据源 Skill 无法覆盖，保留 `Browser_Skill` 抓取财联社快讯（`cls.cn/telegraph`）和华尔街见闻实时要闻（`wallstreetcn.com/live`）作为补充。
- **社交**：调用 `Browser_Skill` 监控**雪球**个股页面讨论热度（帖子数/阅读量变化率）。
- **衍生品**：通过数据源 Skill 获取恒指期货、期权数据（优先 `finance`，降级到 `yahoo-finance` → `akshare-finance`）。

**情绪监控指标**：

| 指标 | 计算方式 | 预警阈值 |
|------|----------|----------|
| 新闻情感得分 | 调用 `NLP_Skill` 对新闻标题和摘要做情感分析，得分范围 [-1, 1] | ≤ -0.6 为突发利空预警 |
| 恒指期货升贴水率 | (期货价 - 现货价) / 现货价 | 贴水 > 0.5% 为恐慌信号 |
| VHSI（恒指波幅指数） | 通过数据源 Skill 获取实时值 | > 25 为恐慌区；> 35 为极端恐慌 |
| Put/Call Ratio | 恒指期权当月合约 Put 成交量 / Call 成交量 | > 1.2 偏空；< 0.7 偏多 |
| 雪球热度变化率 | (今日讨论量 - 5日均) / 5日均 | > 200% 预警关注 |

**NLP 情感分析规则**：
- 对每条新闻/公告标题调用 `NLP_Skill` 进行情感分类：
  - **正面**：盈喜、回购、增持、特别股息、获得重大合同、业绩超预期
  - **负面**：盈警、配售、减持、审计保留意见、被监管调查、做空报告
  - **中性**：须予披露的交易、关联交易、董事变更、例行公告
- 情感得分加权：标题权重 0.4，摘要权重 0.6
- 同一标的 24 小时内多条负面新闻（≥3条）→ 触发"舆情危机"警报

### 2.7 十维基本面评分卡（两种模式共享）

> 每项指标与**恒生二级行业**中位数及自身**历史 5 年分位数**比较打分。满分 100 分。

| # | 维度 | 评分指标 | 权重 | 评分规则 |
|---|------|----------|------|----------|
| 1 | 成长性 | 最新季报营收同比 + 净利润同比 | **15分** | 行业前 20% 分位 = 15 分，后 20% = 0 分，线性插值 |
| 2 | 盈利能力 | ROE-TTM、毛利率-TTM | **12分** | 行业前 20% 分位 = 12 分，后 20% = 0 分，线性插值 |
| 3 | 财务健康 | 资产负债率（越低越好）、流动比率 | **12分** | 优于行业中位数得 6 分，前 20% 得 12 分 |
| 4 | 估值水平 | PE-TTM 历史分位、PB 历史分位 | **12分** | < 20% 分位 = 12 分，> 80% = 0 分 |
| 5 | 股东回报 | 股息率、回购计划 | **10分** | 股息率 > 4% 得 5 分；有回购计划 +5 分 |
| 6 | 管理层动向 | 大股东/高管增减持 | **8分** | 净增持 > 1000 万 HKD = 8 分；净减持 > 1000 万 = 0 分 |
| 7 | 机构共识 | 综合目标价 vs 现价 | **8分** | 上行空间 > 50% = 8 分；> 20% = 4 分；< 0% = 0 分 |
| 8 | 财报质量 | 审计意见 | **8分** | 标准无保留 = 8 分；保留意见 = 2 分；否定/无法表示 = 0 分 |
| 9 | 行业景气 | 行业 PMI/景气指数 | **8分** | 景气上行得 6-8 分，下行得 0-2 分 |
| 10 | 公司治理 | 供股合股历史、关联交易占比、独董比例 | **7分** | 无异常 = 7 分；3年内有供股/合股扣 7 分；每项风险扣 2 分。**各维度得分下限为 0 分，不出现负分** |

**评分使用规则**：
- 总分 **< 40**：**禁止买入**，已持有的标的纳入"加速退出"观察
- 总分 **40-60**：可小仓位试探（观察池仓位上限 8%）
- 总分 **> 60**：可按策略正常仓位操作
- 总分 **> 70**：优质标的，核心池仓位上限 15%
- 评分每周更新一次（周一全量刷新时），重大公告触发即时更新

### 2.8 汇率监控

- 每日 09:00 通过数据源 Skill 获取**离岸人民币兑港币（CNH/HKD）**汇率（优先 `tushare-finance`，降级到 `akshare-finance` → `yahoo-finance`）
- **预警规则**：
  - 单周汇率波动 > 1% → 在收盘总结中提示汇率风险对南向资金收益的影响
  - 单日波动 > 0.5% → 盘中实时监控报告中标注
- **影响评估**：人民币贬值（CNH/HKD 上升）→ 南向资金购买力增强但汇兑损失风险增加

### 2.9 港股特色日历与假期（两种模式共享）

- 通过 `Browser_Skill` 从港交所官网获取年度交易日历（含台风/暴雨停市安排）
- 每日 **08:30** 检查当日是否为交易日：
  - **正常交易日**：执行全部定时任务
  - **非交易日**（周末/公众假期）：自动休眠，仅执行周日《下周展望》
  - **半日市**（农历新年前夕、中秋节前夕、圣诞前夕、元旦前夕；注：以港交所当年公布的交易日历为准，12月31日并非每年都是半日市）：自动调整盘中监控截止时间为 11:55，取消午间复盘，持仓对账提前至 12:30，收盘总结提前至 13:00
  - **台风/暴雨停市**：收到港交所紧急公告后立即进入休眠模式
- 交易日历数据写入记忆系统，避免重复抓取

### 2.10 数据源健康监控

系统持续监控所有数据源 Skill 的可用性状态：

| 状态 | 判定条件 | 系统行为 |
|------|----------|----------|
| 🟢 正常 | 最近 3 次请求均成功，延迟 < 阈值 | 正常使用 |
| 🟡 降级 | 最近 3 次中有 1 次失败或延迟超阈值 | 预警并准备切换到下一优先级 Skill |
| 🔴 不可用 | 连续 3 次请求失败 | 自动切换到优先级链下一个 Skill，每 5 分钟检测恢复 |
| ⚫ 全部不可用 | 某数据维度所有 Skill 均不可用 > 5 分钟 | 触发紧急熔断，暂停信号生成，仅保留止损 |

**恢复策略**：
- 备选 Skill 接管后，每 5 分钟尝试恢复主 Skill 连接
- 主 Skill 恢复后，验证 3 次连续成功请求后自动回切
- 所有切换操作记录日志并在下次报告中汇报
