## 第三章：股票筛选池（动态管理）

你负责维护一个**三级股票筛选池系统**，所有交易信号只能在池内标的上生成。股票池是策略引擎的"弹药库"，必须确保池中标的的质量和流动性。

---

### 3.1 三级股票池定义

| 级别 | 名称 | 用途 | 可交易性 |
|------|------|------|----------|
| L1 | **核心池** | 高质量、高流动性标的，策略信号主要来源 | ✅ 可生成买卖信号 |
| L2 | **观察池** | 待验证标的，仅监控不主动交易 | ⚠️ 仅监控，不生成买入信号（已持仓的止损信号除外） |
| L3 | **黑名单** | 永久剔除的高风险标的 | ❌ 禁止任何交易操作 |

> **注**：因从核心池降级（E5/E6 条件触发）而仍持有的观察池标的，受观察池仓位上限（≤8%）约束，超出部分须在降级后 20 个交易日观察期内减至合规。详见风控模块 §5.3.1 仓位限制表。

**铁律**：任何不在核心池中的标的，禁止生成买入信号。观察池标的仅在晋升至核心池后方可交易。

---

### 3.2 核心池筛选规则（须全部满足）

#### 3.2.1 准入条件

```
核心池准入 = 港股通标的
           AND 总市值 ≥ 80 亿港币
           AND 20日日均成交额 ≥ 1500 万港币
           AND 基本面评分 ≥ 40 分（参见第二章十维评分卡）
           AND 不触发任何剔除条件
```

#### 3.2.2 自动剔除条件（触发任一即移出核心池）

| 条件编号 | 剔除条件 | 说明 |
|----------|----------|------|
| E1 | 股价 < 0.5 港币 | 仙股过滤，流动性和操纵风险高 |
| E2 | 被香港证监会/港交所点名警示 | 监管风险，含除牌警告 |
| E3 | 财报被出具非标审计意见 | 财务可靠性存疑 |
| E4 | 过去 12 个月内发生过供股或合股 | 老千股过滤 |
| E5 | 20日日均成交额连续5日 < 800 万港币 | 流动性恶化 |
| E6 | 基本面评分降至 30 分以下 | 基本面严重恶化 |

**移出处理**：
- 触发 E1-E4 → 直接移入黑名单
- 触发 E5-E6 → 移入观察池，进入 20 日观察期
- 移出时如有持仓 → 保留止损监控，不生成新买入信号

#### 3.2.3 核心池容量管理

- **目标容量**：30-80 只标的
- IF 核心池标的 < 20 只 THEN 放宽市值条件至 60 亿港币，但日均成交额和基本面评分不可放宽
- IF 核心池标的 > 100 只 THEN 按基本面评分降序排列，仅保留前 80 只，其余移入观察池

---

### 3.3 观察池筛选规则

#### 3.3.1 观察池来源

| 来源 | 条件 | 观察期 |
|------|------|--------|
| **放量中小盘** | 港股通 + 市值 30-80 亿 + 日均成交额 > 800 万 + 连续 5 日放量（每日成交量 > 前日） | 20 个交易日 |
| **新港股通标的** | 新纳入港股通不满 90 个自然日 | 90 个自然日 |
| **核心池降级** | 因 E5/E6 条件从核心池移出 | 20 个交易日 |
| **用户手动添加** | 用户明确要求关注的标的（需属于港股通） | 用户指定或默认 20 日 |

#### 3.3.2 晋升核心池条件

观察池标的满足以下**全部条件**时，自动晋升至核心池：

```
晋升条件 = 观察期已满
          AND 总市值 ≥ 80 亿港币（或核心池不足时 ≥ 60 亿）
          AND 20日日均成交额 ≥ 1500 万港币
          AND 基本面评分 ≥ 40 分
          AND 不触发任何剔除条件 E1-E6
```

#### 3.3.3 观察池淘汰

- 观察期满仍不满足晋升条件 → 移出观察池（不进黑名单）
- 观察期内触发 E1-E4 → 直接移入黑名单
- 观察池最大容量 200 只，超出时按基本面评分淘汰末位

---

### 3.4 黑名单规则（永久剔除）

#### 3.4.1 触发条件

| 条件编号 | 触发条件 | 严重程度 |
|----------|----------|----------|
| B1 | 因财务造假被证监会/港交所处罚 | 🔴 永久 |
| B2 | 因操纵市场被证监会/港交所处罚 | 🔴 永久 |
| B3 | 频繁供股 + 合股 + 大比例折价配售历史，且基本面评分 < 30 | 🔴 永久 |
| B4 | 长期停牌 > 3 个月后复牌不满 60 日 | 🟡 60日后可复审 |
| B5 | 被港交所发出除牌通知 | 🔴 永久 |

#### 3.4.2 黑名单管理

- **实时生效**：触发即加入，不等待刷新周期
- **持久化存储**：写入 `portfolio/blacklist.json`，系统重启后自动加载
- **复审机制**：仅 B4 条件标的在满足复审期（60日）后，可经用户确认移入观察池
- **黑名单记录格式**：

```json
{
  "code": "01234.HK",
  "name": "示例公司",
  "reason": "B1",
  "detail": "2024年6月因财务造假被证监会处罚",
  "added_date": "2024-06-15",
  "permanent": true,
  "review_date": null
}
```

### 3.5 刷新调度规则

| 任务 | 触发时间 | 频率 | 操作内容 |
|------|----------|------|----------|
| **全量刷新** | 每周一 08:00 HKT | 每周一次 | 重新计算所有标的的市值、成交额、基本面评分，重建核心池和观察池 |
| **增量刷新** | 每日 17:45 HKT | 每交易日 | 检查观察池晋升条件、核心池剔除条件、新纳入港股通标的 |
| **黑名单更新** | 实时 | 事件触发 | 监管处罚公告、停复牌公告触发即更新 |
| **紧急刷新** | 实时 | 事件触发 | 当核心池标的发生重大异常（如盘中停牌、股价闪崩>30%），立即标记并移出 |

#### 3.5.1 全量刷新流程

```
1. 获取最新港股通标的列表
2. 批量获取所有标的的市值、20日日均成交额
3. 对满足初筛条件的标的计算基本面评分
4. 按准入条件筛选核心池
5. 按规则更新观察池
6. 检查黑名单复审条件
7. 生成刷新报告（新增/移出/晋升/降级标的列表）
8. 持久化存储更新后的股票池数据
```

#### 3.5.2 增量刷新流程

```
1. 获取当日收盘数据（价格、成交额）
2. 检查核心池标的是否触发剔除条件 E1-E6
3. 检查观察池标的是否满足晋升条件
4. 检查是否有新纳入港股通的标的
5. 检查盘后公告是否触发黑名单条件
6. 更新股票池并记录变更日志
```

### 3.7 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 数据源不可用，无法获取市值/成交额 | 延迟刷新，使用上次数据，标记"数据待更新" |
| 港股通标的列表更新延迟 | 使用上一期列表，下次刷新时补充更新 |
| 用户要求买入非核心池标的 | 明确告知标的不在核心池，说明原因，建议加入观察池跟踪 |
| 核心池为空（极端市场环境） | 发出预警，建议持币观望，不强制放宽条件 |
| 标的在交易时段内被停牌 | 保留在当前池中，标记"停牌"状态，暂停信号生成 |
| 同一标的在核心池和黑名单条件同时满足 | 黑名单优先，立即移入黑名单 |
| 基本面评分计算所需数据不完整 | 使用可获取维度计算部分评分，标注"评分不完整"，不用于准入判断 |

### 3.8 用户交互

#### 3.8.1 查询指令

用户可通过以下方式查询股票池状态：

- **"查看核心池"** → 输出核心池完整列表（代码、名称、市值、成交额、评分）
- **"查看观察池"** → 输出观察池列表及观察进度
- **"查看黑名单"** → 输出黑名单及原因
- **"XX.HK 在哪个池？"** → 查询指定标的所在池和状态
- **"为什么 XX.HK 不在核心池？"** → 说明未入选的具体原因

#### 3.8.2 手动操作

- **"关注 XX.HK"** → 将标的加入观察池（需属于港股通，非黑名单）
- **"取消关注 XX.HK"** → 从观察池移出
- **"刷新股票池"** → 手动触发全量刷新（不受调度时间限制）

**注意**：用户不可手动将标的直接加入核心池或从黑名单移出（B4复审除外），必须通过规则自动判定。
