## 第四章：交易策略库（精确量化触发逻辑）

所有策略信号基于实时数据流计算，信号生成时**必须**附带策略名称和全部子条件的达成状态与具体数值。任何策略信号触发后，须同步写入 `signal_snapshots.json` 以支持后续复盘与回溯。

### 策略优先级与冲突仲裁

> 详见第一章 §1.5 策略仲裁优先级规则。
> 当同一标的同时触发多个策略信号时，按以下优先级执行：
> **应急风控 > 强制止损 > 移动止损 > 策略四（极端反转） > 策略三（事件驱动） > 策略一（趋势突破） > 策略二（回调低吸）**

### 过度交易防护

> 详见第一章 §1.6 过度交易防护规则。
> - 单标的每日最多交易 **2 次**（1买1卖或2次调仓）
> - 全账户每日最多 **8 笔**交易指令
> - 止损卖出后该标的进入 **3 个交易日**冷却期

### 策略一：趋势突破策略（适用于强势股 / 主升浪初期）

**适用市场立场**：激进做多、谨慎做多

#### 买入条件（必须全部满足）

1. **形态过滤 — 横盘整理确认**
   - 过去 15 个交易日内，最高价 `HH15` 与最低价 `LL15` 的振幅：`(HH15 - LL15) / LL15 < 12%`
   - 最近 5 个交易日收盘价均在 `MA(20) × 0.97` 与 `MA(20) × 1.03` 之间
   - **记录**：`{ "hh15", "ll15", "amplitude", "ma20", "close_range_5d" }`

2. **放量突破 — 突破确认**
   - 当前收盘价 `P` 首次向上突破 `HH15 × 1.02`（突破区间高点 2% 确认）
   - 成交量比 `VR = V / MA(V,20) > 2.0`（当日成交量 > 20日均量的 2.0 倍）
   - **记录**：`{ "price", "breakout_level", "volume", "vr" }`

3. **技术共振**（须全部满足）
   - `MACD.DIFF` 上穿 `MACD.DEA`，且 `MACD.DIFF > 0`（零轴上方金叉）
   - `收盘价 > MA(20)`，且 `MA(20)` 斜率向上：`MA(20)[T] > MA(20)[T-3]`
   - `RSI(14)` 位于 **55 ~ 80** 区间，且较前日上升：`RSI[T] > RSI[T-1]`
   - **记录**：`{ "macd_diff", "macd_dea", "ma20_slope", "rsi14" }`

4. **资金验证**（至少满足 1 项）
   - 当日大单净流入率 > 0（通过数据源 Skill 获取主力资金流向数据）
   - 最近 5 日南向资金（港股通）累计净买入额 > 0
   - **记录**：`{ "big_order_net_inflow", "southbound_5d_net" }`

5. **预期收益检查**
   - `ATR(14) / P > (交易成本率 + 滑点预算) × 2`
   - 确保日均波动空间足以覆盖双边成本
   - **记录**：`{ "atr14", "cost_ratio", "slippage", "profit_check_passed" }`

#### 卖出条件（任一触发即执行）

| 条件 | 触发规则 | 执行方式 |
|------|---------|---------|
| **趋势失败** | 收盘价跌破盘整区间下沿 `LL15 × 0.98`（注：此处 `LL15` 使用买入时信号快照中锁定的值，非实时滚动计算值，以确保止损位在持仓期间保持稳定） | 次日开盘市价清仓 |
| **指标反转** | MACD 死叉（DIFF 下穿 DEA）**或** RSI(14) 从 70 以上跌破 65 | 当日限价单卖出 |
| **量价背离** | 股价创 20 日新高，但 RSI 或 OBV 未创新高，且当日成交量 < 突破日成交量 × 50% | 减仓 50%，剩余设移动止损 |
| **时间止损** | 买入后 **10 个交易日**内涨幅未达 5%，且 MA(5) 斜率走平（`abs(MA5[T]-MA5[T-3])/MA5[T-3] < 0.5%`） | 减仓 50% |

### 策略二：回调低吸策略（适用于确认上升趋势的回踩）

**适用市场立场**：激进做多、谨慎做多

#### 买入条件（必须全部满足）

1. **大趋势确认**（周线级别）
   - `周收盘价 > 周 EMA(20) > 周 EMA(60)`
   - 周 `EMA(20)` 连续 **3 周**向上：`EMA20_W[T] > EMA20_W[T-1] > EMA20_W[T-2]`
   - **记录**：`{ "weekly_close", "weekly_ema20", "weekly_ema60", "ema20_trend_weeks" }`

2. **回调到位**（日线级别）
   - 日线收盘价**从上方**首次回调至 `MA(30)` 或 `MA(60)` 的 **±1%** 范围内
   - 当日 K 线出现**看涨反转形态**（量化定义）：
     - **锤子线**：下影线 ≥ 实体 × 2，上影线 ≤ 实体 × 0.3，实体位于 K 线上部 1/3
     - **看涨吞没**：当日阳线实体完全吞噬前一日阴线实体，且当日收盘 > 前日开盘
   - **记录**：`{ "close", "ma30", "ma60", "distance_to_ma", "reversal_pattern" }`

3. **动能衰竭确认**
   - `RSI(14)` 在 **35 ~ 45** 区间（超卖但未极端）
   - `KDJ.J` 值 < 20 并向上拐头：`J[T] > J[T-1]` 且 `J[T-1] < 20`
   - **记录**：`{ "rsi14", "kdj_j", "kdj_j_prev", "j_turning_up" }`

4. **缩量企稳**
   - 最近 3 日成交量逐日萎缩：`V[T] < V[T-1] < V[T-2]`
   - 今日成交量 `V < MA(V,10) × 0.6`
   - **记录**：`{ "vol_3d": [V_T, V_T1, V_T2], "vol_ma10", "vol_shrink_ratio" }`

#### 卖出条件（任一触发即执行）

| 条件 | 触发规则 | 执行方式 |
|------|---------|---------|
| **压力位受阻** | 反弹至前期高点 ±1% 或涨幅超本次回调幅度 80%，出现滞涨信号（长上影线 > 实体 × 2 或看跌吞没） | 减仓 50%，剩余挂移动止损 |
| **趋势破位** | 收盘价跌破 `MA(60) × 0.99` | 次日开盘清仓 |
| **时间止损** | 买入后 **15 个交易日**内未反弹至回调起点的 50% | 减仓 50% |

### 策略三：事件驱动策略（财报 / 公告 / 回购）

**适用市场立场**：激进做多、谨慎做多、中性观望（仅限强事件驱动）

#### 3A. 业绩超预期买入

**条件（全部满足）**：

1. **财报数据验证**
   - 营收同比增长 > **20%**
   - 净利润同比增长 > **25%**
   - 毛利率同比提升 > **2 个百分点**（绝对值差，即 `earnings_margin_improvement_min = 0.02` 表示从 30% 提升至 32%，而非 30% × 1.02）
   - **记录**：`{ "revenue_yoy", "net_profit_yoy", "gross_margin_change" }`

2. **市场反应确认**
   - 公告后首个交易日跳空高开 > **4%**
   - 成交量比 `VR > 2.0`
   - **记录**：`{ "gap_up_pct", "vr" }`

3. **基本面门槛**
   - 十维基本面评分 ≥ **50**
   - **记录**：`{ "fundamental_score" }`

**执行方式**：
- 公告后次日开盘，以前 15 分钟 VWAP 为参考价
- 限价单（VWAP + 0.5% 以内）介入
- 30 分钟内未成交则**自动撤单**

#### 3B. 回购/增持买入

**条件（任一满足即可触发）**：

| 事件类型 | 触发条件 | 补充要求 |
|---------|---------|---------|
| **回购计划** | 正式公告回购金额占市值 > **1.5%**，且回购价上限 > 当前价 × **1.1** | 公告后 3 日内股价偏离公告日收盘价 ±4% 以内 |
| **股东/高管增持** | 增持金额 > **800 万港币** | 公告后 3 日内股价偏离公告日收盘价 ±4% 以内 |

> **注**：此处增持触发阈值（800 万港币）用于事件驱动信号触发，低于基本面评分卡中的 1000 万港币阈值（见 §2.7 第8维"董监高增持"），两者适用场景不同——前者为短期事件驱动买入条件，后者为中长期基本面评分维度。详见 `strategy_params.json` 中 `insider_buy_min_amount_hkd` 与 `fundamental_scoring.insider_buy_threshold_hkd` 的 `_note` 说明。

**执行方式**：
- 条件满足后次日开盘介入
- **记录**：`{ "event_type", "buyback_amount", "buyback_ratio", "buyback_price_cap", "current_price", "price_deviation" }`

#### 策略三统一卖出条件（任一触发）

| 条件 | 触发规则 | 执行方式 |
|------|---------|---------|
| **利好兑现** | 跳空后 3 日无法创新高，且 RSI > 80 | 限价卖出 |
| **缺口回补** | 10 个交易日内完全回补跳空缺口，且收盘价跌破缺口下沿 1% | 立即止损清仓 |
| **回购未执行** | 公告后 20 个交易日，公司实际回购金额 < 计划的 20% | 减仓 50% |

### 策略四：极端情绪反转策略（逆向抄底）

**适用市场立场**：激进做多、谨慎做多（需满足额外条件：恒指 RSI(14) < 28）、中性观望、持币防守时可使用。持币防守时为**唯一**可用策略，仓位减半；其他适用立场时作为对冲/防御补充

> ⚠️ **高风险策略**：此策略逆势操作，须严格遵守分批建仓和止损纪律。

#### 买入条件（必须全部满足）

1. **短期超卖 — 技术极端**
   - `RSI(14) < 18`（极度超卖）
   - `KDJ.J < -5`（J 值为负数，极端超卖）
   - **记录**：`{ "rsi14", "kdj_j" }`

2. **恐慌抛售 — 量价确认**
   - 连续 3 日累计跌幅 > **15%**
   - 后两日成交量均 > `MA(V,20) × 1.5`（放量恐慌）
   - **记录**：`{ "cum_decline_3d", "vol_day2", "vol_day3", "vol_ma20" }`

3. **基本面安全底线 — 非老千股**
   - 最新季度**扣非净利润 > 0**（非亏损企业）
   - `PE-TTM 历史分位 < 15%`（估值处于历史极低位）
   - 十维基本面评分 ≥ **50**
   - 该标的不在黑名单中，过去 12 个月无供股/合股记录
   - **记录**：`{ "deducted_net_profit", "pe_ttm_percentile", "fundamental_score", "blacklist_check", "dilution_check" }`

4. **大盘环境配合**
   - 恒生指数 `RSI(14) < 28`
   - 恒指处于关键支撑位：前低点 ±2% 范围内，或 `MA(120)` 附近
   - **记录**：`{ "hsi_rsi14", "hsi_support_level", "hsi_distance_to_support" }`

#### 分批建仓执行

```
┌─────────────────────────────────────────────────────────┐
│                    策略四：分批建仓流程                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  信号触发当日                                              │
│  └─ 15:30 后检查（半日市调整为 11:45）                       │
│     <!-- 注：11:45 为半日市（12:00 收盘）的尾盘检查时间，   │
│      收盘前 15 分钟更贴近"尾盘"含义。 -->                    │
│      │                                                   │
│      ├─ 股价较日内高点回落 > 2%？                            │
│      │   ├─ YES → 第一批 40% 限价买入                       │
│      │   └─ NO  → 放弃本次信号                              │
│      │                                                   │
│  次日开盘                                                  │
│  ├─ 低开 > 3%？                                           │
│  │   ├─ YES → 取消第二批，评估第一批止损                      │
│  │   └─ NO  → 收盘价 > 昨日收盘（收阳线）？                   │
│  │       ├─ YES → 第二批 60% 以 VWAP 附近限价买入            │
│  │       └─ NO  → 暂缓，观察至第三日再决定                    │
│  │                                                       │
└─────────────────────────────────────────────────────────┘
```

- **第一批 40%**：信号触发当日尾盘，确认股价较日内高点回落 > 2%（排除 V 型反转追高风险），限价买入
- **第二批 60%**：次日若收盘价 > 今日收盘价（收阳线确认），开盘后前 15 分钟 VWAP 附近限价买入
- **取消条件**：次日若低开 > 3%，取消第二批，并对第一批执行止损评估

#### 卖出条件（任一触发即执行）

| 条件 | 触发规则 | 执行方式 |
|------|---------|---------|
| **反弹受阻** | 股价反弹至 MA(10) 或 MA(20) 附近，连续 2 日收盘无法站稳均线上方，且 RSI 回升至 55 以上后拐头向下 | 限价卖出清仓 |
| **分步止盈** | 自买入均价反弹 > **12%** | 止盈 50%，剩余仓位改用移动止损（回撤 5%）跟踪 |

### 信号快照记录规则

每当任何策略生成买入或卖出信号时，**必须**同步将以下信息写入 `portfolio/signal_snapshots.json`：

```json

  "timestamp": "ISO 8601 格式",
  "stock_code": "标的代码",
  "stock_name": "标的名称",
  "signal_type": "BUY | SELL",
  "strategy": "strategy_1 | strategy_2 | strategy_3a | strategy_3b | strategy_4",
  "market_stance": "当前市场立场",
  "conditions": {
    "condition_1": {
      "name": "条件名称",
      "required_value": "阈值",
      "actual_value": "实际值",
      "passed": true
    }
  },
  "price_at_signal": 0.00,
  "suggested_action": {
    "order_type": "LIMIT | MARKET",
    "price": 0.00,
    "quantity_pct": "仓位百分比",
    "stop_loss": 0.00,
    "take_profit": null
  },
  "data_source": "信号计算所用数据来源",
  "confidence_factors": {
    "conditions_met": "5/5",
    "data_freshness": "延迟秒数",
    "cross_validated": true
  }
}
```

**快照管理规则**：
- 快照中的 `ll15`、`hh15` 等价格字段在卖出判定时作为**固定参考值**使用（即买入时锁定的值），不随持仓期间的滚动窗口变化而更新，以保证止损/止盈位的确定性
- 每条信号生成一条快照记录，追加写入（不覆盖）
- 快照保留最近 **90 个交易日**的数据，超期自动归档。归档细节：超过 90 个交易日的快照在每日盘后数据更新（17:30）时自动迁移至 `portfolio/{user}/archive/signal_snapshots_{YYYY}.json`；归档数据仍可被月度优化报告（§7.6）引用
- 卖出信号快照中须关联对应的买入快照 ID（`related_buy_snapshot_id`）
- 每日复盘（17:00）和周度评估（周五）均依赖快照数据进行信号回溯

### 通用规则

1. **所有策略信号**须经过第五章风控模块的仓位检查后方可输出为正式指令单
2. **同一标的**不得同时持有来自不同策略的多头仓位（后触发的信号自动归入已有仓位管理）
3. **信号有效期**：买入信号在触发后 **2 个交易日**内有效，逾期自动作废
4. **数据不足降级**：若计算某策略所需的技术指标因历史数据不足（如上市不满 120 日无法计算 MA120），该策略对该标的自动跳过，不生成信号
5. **策略暂停恢复**：被自动暂停的策略（连续20笔胜率不达标），须在第七章绩效优化模块中经参数调优或用户手动确认后方可恢复

### 4.7 边界情况处理

| 场景 | 处理规则 |
|------|----------|
| 同一标的同时触发买入和卖出信号 | 立即暂停该标的所有操作，发出警报："检测到 {stock_code} 存在买卖信号矛盾（{买入策略} vs {卖出原因}），已暂停操作，等待下一监控周期重新计算" |
| 多个策略同时触发同一标的买入 | 按策略优先级仲裁（策略四 > 策略三 > 策略一 > 策略二），仅执行最高优先级策略的信号，其余记录为"被仲裁覆盖" |
| 策略计算所需数据不足 | 若因上市时间短（如 < 120 日无法计算 MA120）或数据源缺失导致某些指标无法计算，该策略对该标的自动跳过，在观察列表中标注"数据不足，暂不评估" |
| 策略信号在非交易时段触发 | 盘前/盘后产生的信号标记为"待确认"，在下一交易日开盘前的盘前报告中展示，开盘后验证条件是否仍然成立 |
| 所有四大策略均被暂停（衰退） | 紧急通知用户："所有量化策略因持续表现不佳已暂停，系统仅运行止损和应急风控。建议等待月度优化报告或手动恢复某个策略。" |
| 信号快照文件写入失败 | 信号仍然正常输出为指令单（快照是辅助记录，不阻塞核心流程），在下次报告中提示"部分信号快照未能保存，复盘分析可能不完整" |
| 事件驱动策略公告解读不确定 | 当 NLP 情感分析置信度 < 60% 时，不自动触发策略三信号，改为在观察列表中标注"待人工确认"，并提示用户："检测到 {stock_code} 发布公告，但内容解读存在不确定性，请您确认后告知是否执行" |
