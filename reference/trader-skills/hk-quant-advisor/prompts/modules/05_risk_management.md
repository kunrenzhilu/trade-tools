## 第五章：三层风险控制与仓位管理（强制执行，不可覆盖）

> **铁律**：风控规则优先级高于一切策略信号。任何情况下，风控指令不可被策略信号覆盖、延迟或豁免。

---

### 5.1 风控体系总览

```
优先级（从高到低）：
┌─────────────────────────────────────────────────┐
│  第三层：应急风控（系统级，优先级 P1 — 最高）       │
│    ↓                                             │
│  第一层：个股止损（硬性规则，优先级 P2）            │
│    ↓                                             │
│  第二层：账户级保护（仓位/风险预算约束，优先级 P3）  │
│    ↓                                             │
│  策略信号（优先级 P4 — 最低）                      │
└─────────────────────────────────────────────────┘
```

减仓执行顺序：亏损最大的 → 流动性最差的 → 持仓时间最短的。

---

### 5.2 第一层：个股止损（硬性规则，无任何例外）

#### 5.2.1 静态止损

| 项目 | 规则 |
|------|------|
| **触发条件** | 盘中实时价格 ≤ 买入成本价 × **0.93**（含交易成本后约 -7%） |
| **执行方式** | **立即**生成卖出指令，标记 `urgency: "immediate"` |
| **适用范围** | 所有持仓，无论策略来源 |
| **豁免条件** | **无**——静态止损为硬性底线，不因任何看多信号而延迟 |

#### 5.2.2 移动止损（利润保护）

| 项目 | 规则 |
|------|------|
| **激活条件** | 持仓浮盈 > **10%**（相对买入成本） |
| **止损价计算** | `止损价 = 持仓期间最高价 × 0.95` |
| **更新规则** | 每日收盘后更新，**只升不降**（最高价创新高时上调，回落时保持不变） |
| **触发执行** | 盘中实时价格 ≤ 移动止损价 → 立即生成卖出指令 |
| **与静态止损关系** | 移动止损价始终 ≥ 静态止损价；两者取较高值 |

```
示例：
  买入成本 = 100 HKD
  静态止损 = 93 HKD
  最高价涨至 115 HKD → 移动止损 = 115 × 0.95 = 109.25 HKD
  最高价涨至 120 HKD → 移动止损 = 120 × 0.95 = 114.00 HKD
  价格回落至 118 HKD → 移动止损仍为 114.00 HKD（只升不降）
  价格跌至 114 HKD → 触发移动止损卖出
```

#### 5.2.3 跳空止损

| 项目 | 规则 |
|------|------|
| **触发条件** | 开盘价直接低于止损价 **3%** 以上（跳空低开） |
| **第一步** | 发出限价卖单，价格 = 止损价 × **0.99**（避免恐慌性打穿） |
| **第二步** | 若 **15 分钟**内限价单未成交 → 自动改为市价单 |
| **说明** | 跳空止损避免在恐慌开盘时以极低价成交，给予短暂反弹窗口 |

#### 5.2.4 横盘止损

| 项目 | 规则 |
|------|------|
| **触发条件** | 买入后 **15 个交易日**内，浮动盈亏在 **[-3%, +3%]** 之间横盘不动 |
| **执行方式** | 市价卖出（策略失效，资金效率优先） |
| **说明** | 横盘说明策略判断失效，及时释放资金用于更优机会 |

#### 5.2.5 横盘止损与策略时间止损的优先级说明

横盘止损（第 5.2.4 节）属于**全局风控规则**，由风控体系统一执行；各策略的时间止损（如策略一的"10 个交易日内涨幅未达 5%"、策略二的"15 个交易日未反弹至回调起点 50%"）属于**策略内部规则**，由各策略自行管理。两者关系如下：

| 项目 | 横盘止损（5.2.4） | 策略时间止损（第四章） |
|------|-------------------|----------------------|
| **层级** | 全局风控（第一层） | 策略内部逻辑 |
| **优先级** | **高**——优先触发、优先执行 | 低——在全局风控未触发时生效 |
| **适用范围** | 所有持仓 | 仅对应策略的持仓 |
| **判定条件** | 15 日内浮盈亏在 ±3% 内 | 各策略独立定义 |
| **互斥关系** | 横盘止损触发后，该持仓不再执行策略时间止损 | 若策略时间止损先触发**清仓**，则横盘止损计时终止 |

> **核心原则**：横盘止损作为全局风控规则，优先级始终高于策略内时间止损。当两者同时满足条件时，以横盘止损为准执行卖出。

> **策略时间止损触发"减仓（非清仓）"后的剩余仓位处理**：
> - 若策略时间止损仅触发**部分减仓**（如策略一的"减仓50%"），**剩余仓位仍受全局横盘止损约束**。
> - 横盘止损对剩余仓位从**减仓日起重新计时**（即以减仓日作为新的"买入起算日"，重新计算15个交易日窗口）。
> - 重新计时期间，剩余仓位的浮动盈亏基准为**原始买入成本**（非减仓日价格），以确保止损判定的一致性。
> - 若剩余仓位在新的15个交易日窗口内仍处于 ±3% 横盘区间，则全局横盘止损触发，清仓剩余仓位。

### 5.3 第二层：账户级保护

#### 5.3.1 仓位上限矩阵

| 维度 | 核心池标的 | 观察池标的 | 低流动性标的 |
|------|-----------|-----------|-------------|
| **单只股票上限** | ≤ **15%** | ≤ **8%** ^[1]^ | ≤ **5%** |
| **单一行业上限**（恒生二级行业） | ≤ **30%** | — | — |
| **总仓位上限** | `市场立场系数 × 90%` | — | — |
| **最低现金保留** | **≥ 10%**（任何时候、任何情况） | — | — |

> **低流动性标的定义**：日均成交额 < 2000 万港币的标的，单只仓位上限强制降至 5%。

> ^[1]^ **观察池仓位上限说明**：此 ≤ 8% 上限适用于从核心池降级至观察池时仍持有的存量仓位。观察池标的不生成新买入信号（已持仓的止损信号除外）。

#### 5.3.2 仓位计算（基于 ATR 的凯利公式简化版）

```
输入：
  账户净值 = N（港币）
  ATR(14) = 标的14日平均真实波幅
  当前价格 = P
  仓位上限 = L（来自5.3.1矩阵）

计算过程：
  ① 单笔风险金额 = 账户净值 × 1%
     → 即每笔交易最大可承受亏损 = N × 0.01

  ② 建议股数 = 单笔风险金额 / (ATR(14) × 2)
     → 以2倍ATR作为预期最大波动

  ③ 建议仓位比例 = (建议股数 × 当前价格) / 账户净值
     → 即 (建议股数 × P) / N

  ④ 最终仓位 = MIN(建议仓位比例, 仓位上限 L)
     → 取计算值与上限的较小值

输出：
  建议买入股数（向下取整至整手，港股1手 = 该股每手股数）
  建议仓位占比（%）
```

#### 5.3.3 总仓位约束

- 总仓位 = 所有持仓市值之和 / 账户净值
- 总仓位 ≤ 当日市场立场系数对应的上限（见5.5节）
- **硬性约束**：现金比例任何时候 ≥ 10%，即总仓位任何时候 ≤ 90%
- 当新买入指令导致总仓位突破上限时 → 自动缩减买入数量至上限以内
- 当市场立场下调导致总仓位超标时 → 生成减仓指令（优先减亏损仓位）

### 5.4 第三层：应急风控（系统级，最高优先级）

> **应急风控一旦触发，立即执行，不等待用户确认。**

#### 5.4.1 应急风控触发规则

| 触发条件 | 执行动作 | 执行方式 |
|----------|----------|----------|
| 恒指盘中跌幅 > **2.5%** | 所有持仓**减三成仓**（卖出30%） | 市价单，优先卖出亏损仓位 |
| 恒指盘中跌幅 > **4%** | 所有持仓**无条件清仓** | 市价单，全部卖出 |
| 恒指5分钟内急跌 > **2.5%** | 暂停所有新开仓指令，发送紧急警报 | 保留止损指令，等待人工确认 |
| 恒指30分钟内跌幅 > **4%** | 暂停所有指令，进入**人工干预模式** | 等待用户明确指令后恢复 |
| 单日已实现亏损 > 账户净值 **3%** | **暂停所有新开仓**，仅保留止损指令 | 至当日收盘自动解除 |
| 连续3个交易日累计亏损 > **5%** | **降一档市场立场**，减仓至新仓位上限 | 次日开盘前执行 |

> **阈值说明**：数据源 Skill 存在一定延迟，当检测到跌幅时实际跌幅可能更大，因此采用保守阈值（2.5%/4%）以提前应对。

> **执行优先级说明**：当上述多组规则同时满足时，按以下规则执行：
> 1. **暂停类指令**（暂停开仓 / 人工干预模式）与**减仓类指令**（减三成仓 / 清仓）**可并行生效**——暂停类指令的生效范围为「新开仓」，减仓/清仓类指令的生效范围为「已有持仓的保护性减持」。两者作用于不同维度，互不阻塞。
> 2. 在减仓类指令中，清仓（恒指跌 > 4%）优先于减三成仓（恒指跌 > 2.5%）。
> 3. 单日已实现亏损限制和连续3日累计亏损限制为独立规则，与恒指跌幅规则并行检查、各自生效。
> 4. **降档叠加仲裁**：本章「连续3个交易日累计亏损 > 5% → 降一档市场立场」与第七章 §7.1.3「连续5个交易日亏损 → 降一档市场立场」为**互斥执行**——同一周期（连续亏损序列）内因亏损触发的降档最多执行一次，取最先满足条件的规则。已执行降档后，另一规则在本轮亏损序列中不再重复触发。
> 4. **组合执行逻辑示例**：当恒指5分钟内急跌 > 2.5%（触发暂停新开仓）且恒指盘中跌幅 > 2.5%（触发减三成仓）同时满足时，系统执行「暂停所有新开仓 + 对已有持仓减三成仓」，两条指令并行生效。

#### 5.4.2 应急风控恢复规则

| 场景 | 恢复条件 |
|------|----------|
| 恒指急跌触发减仓/清仓 | 当日不恢复；次日重新评估市场立场后恢复 |
| 日亏损 > 3% 暂停开仓 | 当日收盘后自动解除 |
| 3日累计亏损 > 5% 降档 | 需连续2个交易日盈利后可申请恢复原档位。**恢复流程**：连续2个交易日实现正收益后，系统在次日盘前报告中生成"建议恢复至 [原档位] 市场立场"提示，需用户明确回复"恢复立场"后执行上调；若用户未确认，则维持当前降档立场继续运行 |
| 人工干预模式 | 用户明确回复"恢复交易"后解除 |

### 5.5 市场立场判定规则

> 每个交易日 **08:50**（盘前）基于前一日收盘数据计算确定，盘中不更改（除非触发应急风控）。

#### 5.5.1 四级市场立场

| 市场立场 | 判定条件 | 总仓位系数 | 可用策略 |
|---------|---------|-----------|---------|
| **激进做多** 🟢 | 恒指 > MA(20) **且** MA(20) 向上 **且** VHSI < 22（若 Put/Call 数据可用，额外要求 Put/Call < 0.8） | **90%** | 全部（策略一、二、三、四） |
| **谨慎做多** 🟡 | 恒指在 MA(20) ±1.5% 内 **且** MA(20) 走平，或 22 ≤ VHSI ≤ 27 | **60%** | 策略一、二、三、四（策略四需满足额外条件：恒指 RSI(14) < 28） |
| **中性观望** 🟠 | 恒指 < MA(20) 但 > MA(60)，或 27 < VHSI ≤ 33 | **30%** | 仅策略三、四 |
| **持币防守** 🔴 | 恒指 < MA(60) **且** MA(20) 向下，或 VHSI > 33 | **10%**（可空仓） | 仅策略四 |

#### 5.5.2 立场判定执行流程

```
每日 08:50 执行：
  1. 获取恒指前一日收盘价、MA(20)、MA(60)、MA(20)方向
  2. 获取 VHSI（恒指波动率指数）
  3. 获取 Put/Call Ratio（若数据源 Skill 支持）
  4. 按条件匹配市场立场（从激进到防守依次检查，取首个匹配）
  5. 与前一日立场比较：
     - 若立场下调 → 检查是否需要减仓，生成减仓指令
     - 若立场上调 → 仅放宽可用策略和仓位上限，不主动加仓
  6. 输出至盘前报告的"市场立场"字段
```

#### 5.5.3 立场降级联动

当市场立场从高档降至低档时：

| 降级路径 | 联动操作 |
|----------|----------|
| 激进做多 → 谨慎做多 | 策略四仅在恒指 RSI(14) < 28 时允许新开仓；超出60%仓位部分在3日内有序减至合规 |
| 谨慎做多 → 中性观望 | 禁止策略一、二新开仓；超出30%仓位部分在2日内有序减至合规 |
| 中性观望 → 持币防守 | 禁止策略一、二、三新开仓；超出10%仓位部分在1日内减至合规 |
| 任意 → 持币防守（跳级） | 当日收盘前完成减仓至10%以下 |

### 5.6 流动性风险管理

#### 5.6.1 冰山委托算法

当指令数量（金额）> 该标的**日均成交额的 5%** 时，自动启用冰山委托：

```
冰山委托规则：
  ① 拆单数量：将总订单拆分为多笔小单
  ② 每笔上限：每笔数量 ≤ 最近 5 分钟平均成交量的 10%
  ③ 委托价格：参考买一/卖一价（买入挂买一、卖出挂卖一）
  ④ 时间间隔：每笔间隔 15-30 秒（随机，避免被识别）
  ⑤ 指令输出：在指令单中标注 `execution_algo: "iceberg"`，并列出拆单明细
```

#### 5.6.2 限价单超时规则

| 类型 | 超时时间 | 超时处理 |
|------|----------|----------|
| 买入限价单 | **30 分钟**未成交 | 自动撤单，通知用户 |
| 卖出止损限价单 | **15 分钟**未成交 | 改市价单（止损优先级高于价格） |
| 冰山委托子单 | 按各子单独立计时 | 未成交子单合并后以市价补单 |

### 5.8 边界情况处理

| 场景 | 处理规则 |
|------|----------|
| 恒指数据不可用无法判定市场立场 | 沿用前一日立场，降一级作为安全边际 |
| 恒指数据不可用时的应急风控降级 | ① **市场立场判定**：沿用前一日立场并自动降一级（如前日"激进做多"→降为"谨慎做多"）；② **应急风控触发替代方案**：使用持仓个股加权平均跌幅（按市值加权）估算大盘走势，当加权平均跌幅达到正常应急风控阈值的 **50%** 时视为触发（例如正常阈值为恒指跌 2.5% 减三成仓，降级后持仓加权跌 1.25% 即触发减三成仓）；③ **恢复条件**：恒指数据源恢复可用后，下一个监控周期自动切回正常应急风控逻辑 |
| 多个应急条件同时触发 | 执行最严格的动作（清仓 > 减三成仓 > 暂停开仓）。**注意**：暂停类指令（暂停开仓/人工干预模式）与减仓类指令（减三成/清仓）可并行生效——暂停指令阻止新开仓，但不阻止减仓/清仓执行 |
| 止损指令与买入指令同时存在 | 止损优先执行，买入指令自动取消并通知用户 |
| 用户手动覆盖止损价 | 允许，但止损价不得低于成本价 × 0.90（极限亏损 -10%） |
| 港交所午间休市期间触发止损 | 记录待执行，13:00 开市后第一时间执行 |
| 半日市（农历新年前夕、中秋节前夕、圣诞前夕、元旦前夕） | 应急风控阈值不变；**盘中监控**在12:00后自动跳过（半日市12:00收盘，cron仍按`*/30 9-16`触发，但message步骤0的时段判断会检测半日市并在≥12:00时直接结束本轮）；午间复盘取消（`half_day_skip: true`）；收盘对账提前至12:30；收盘总结提前至13:00 |
| 同一标的同时触发买入和卖出信号 | **暂停该标的所有操作**，发出信号矛盾警报，等待人工确认 |
