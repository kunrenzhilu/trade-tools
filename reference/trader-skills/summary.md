## 二、各 Skill 核心思路深度解析

### 1. `hk-quant-advisor` — 严格规则驱动的量化系统

**设计哲学**：把人类交易员的判断"代码化"——不是给建议，而是生成可执行的标准化指令单。

**核心架构（9层模块）**：
- 角色定义：三重角色合一（交易员 + 风控官 + 绩效分析师）
- 数据层：多源获取 + 自动降级（腾讯 → Yahoo → AKShare，付费优先）
- 股票池管理：核心池（主动交易）/ 观察池 / 黑名单 三分法
- **四大量化策略**（硬性条件，不允许主观覆盖）：
  - 策略一：趋势突破（横盘整理后放量突破，MACD零轴上方金叉 + RSI 55~80 + 资金验证）
  - 策略二：回调低吸（周线上升趋势中，回调至MA30/MA60附近出现K线反转形态 + 缩量企稳）
  - 策略三：事件驱动（业绩超预期 + 跳空高开验证；或回购/增持超阈值）
  - 策略四：极端情绪反转（RSI<18 + KDJ.J<-5 + 连续3日跌>15% + 大盘恒指RSI<28，分批建仓）
- **三层风控**（硬性规则，优先级高于所有策略）：
  - 第一层：个股止损（静态-7% / 移动止损 / 跳空止损 / 横盘15日止损）
  - 第二层：账户级仓位矩阵（单股≤15% / 单行业≤30% / 现金≥10%；用ATR凯利公式计算仓位）
  - 第三层：应急风控（恒指盘中跌>2.5%减三成，跌>4%清仓）
- 绩效闭环：自动记录信号快照 → 复盘 → 参数自优化 → 月度策略优化报告

**核心思路总结**：这是一套**规则优先、数据驱动、自我进化**的系统。所有决策必须通过量化条件触发，没有主观判断空间。它的策略设计注重"买入条件全部满足才行动"，风控设计采用"任一触发立即执行"。

### 2. `westock-data` — 纯数据获取工具

**设计哲学**：只提供数据，不提供判断。定位是"查询员"而不是"分析师"。

**核心能力**：
- 实时行情、K线（日/周/月，最大2000条）、分时
- 财务三大报表（多期对比，A股/港股/美股）
- 资金流向（A股主力大单/港股南向/美股卖空）
- 技术指标（MA/MACD/KDJ/RSI/BOLL/WR/DMI，支持历史区间）
- 筹码成本（仅A股）、股东结构、机构评级、一致预期
- 宏观经济（GDP/CPI/PPI/PMI等）
- 市场资讯、公告、研报全文

**核心思路总结**：**数据基础设施**。它的存在价值是为其他skill（尤其是 `hk-quant-advisor` 和 `westock-partner`）提供数据支撑。独立使用时相当于一个增强版的股票数据API客户端，返回结构化Markdown表格，让AI直接读取分析。

---

### 3. `westock-tool` — 多维度选股筛选工具

**设计哲学**：从"全市场"出发，按条件缩小候选池。与 `westock-data` 是"选股"与"查股"的互补关系。

**三种选股方式**：
- **条件选股**（`filter`）：自定义表达式，支持 `intersect()`/`union()` 逻辑组合，覆盖价格/估值/财务/资金等字段
- **策略选股**（`strategy`）：50+预置策略（技术信号类：MACD金叉/KDJ金叉；基本面类：高股息/高ROE；大师策略：巴菲特/格雷厄姆/费雪；资金类：主力抢筹/机构接盘）
- **标签选股**（`label`）：90+预定义标签（股东属性：央企/国企/陆股通；风险：ST/破发；估值：破净；财务特征：高ROE/业绩预增等）

**核心思路总结**：**量化选股的入口工具**。适合构建"选股漏斗"——先用条件筛出候选池，再用 `westock-data` 做深度尽调。策略选股提供了现成的量化信号（MACD/RSI等），标签选股适合做基本面初筛。

---

### 4. `westock-partner` — 多流派AI圆桌炒股顾问

**设计哲学**：把不同投资流派的方法论"人格化"，用圆桌讨论形式展示思维碰撞，把选择权还给用户。

**六位高手的方法论框架**：

| 高手 | 流派 | 核心逻辑 | 操作周期 |
|------|------|---------|---------|
| 🌊 星姐 | 产业趋势派 | 政策→产业链→核心卡位（资源涨价买有矿/扩产买受益环节/路线未定买铲子），估值做硬约束 | 中长线 |
| 📡 洲仔 | 信号派 | 政策→产业→资讯→资金**四层对齐**才动手，任一层变坏就减仓 | 中线（周/月） |
| 🧮 文仔 | 估值派 | Price=EPS×PE，先看增速选工具（高增速用PEG/低增速用PE Bands+EPS斜率），横向+纵向估值对比 | 中长线 |
| 🏔️ 坤仔 | 逆向派 | 只买大市值公司，K线支撑位分10次建仓，等牛市力竭或PE历史高位卖出 | 长线 |
| 🔬 钊仔 | 基本面派 | 生活体感→数据验证→管理层靠谱度，按季度跟踪，永远不空仓（熊市换防御资产） | 长线（满仓） |
| ⚡ 磊仔 | 短线派 | 从新闻抓一周持续主线，选大市值龙头（100亿+），9:25集合竞价确认，持3-5天，破5/10日线立走 | 超短线 |

**关键设计亮点**：
- 强制多人分工（方向选择类问题必须3-4人，防止单人包圆）
- 数据铁律（先查后写，禁止用模型记忆补数据，关键数字交叉验证）
- 能力边界严守（星姐≠全能研究员；文仔不给短线时机；磊仔不讲估值深度）
- 第零步宏观扫描（联网获取当天新闻，不用模型记忆）

**核心思路总结**：这不是一个给答案的工具，而是一个**展示思维过程**的工具。它的价值在于让用户看到不同框架如何解读同一只股票，从而建立自己的判断依据。

---

### 5. `MX_StockPick` — 自然语言驱动的AI智能选股

**设计哲学**：降低选股门槛，用自然语言代替技术条件表达式。

**核心能力**：
- 后端：东方财富妙想大模型，能理解模糊查询（"科技龙头"、"苹果产业链"）
- 覆盖最广：A股/港股/美股/基金/ETF/可转债/板块
- 支持复合逻辑（且/或/非/排序）
- A股进阶：龙虎榜/高管信息/分红并购/券商金股
- 输出CSV文件（含中文列名说明），适合进一步量化处理

**局限**：
- 需要付费API（东方财富妙想 `EM_API_KEY`）
- 自然语言查询的精确性依赖大模型解析，存在歧义
- 不内置标准化的策略信号（不像 `westock-tool` 有明确的MACD金叉等规则）
- 缺乏风控模块

**核心思路总结**：**NLP驱动的灵活选股工具**。适合快速探索、模糊概念选股（如"新能源产业链"），或需要大量数据做进一步分析的场景（输出CSV）。但它的"智能"是基于大模型理解，而非规则引擎，对于需要精确量化条件的场景不够可控。

## 三、横向对比：相同与不同

### 3.1 核心共同点

| 共同点 | 说明 |
|--------|------|
| **数据驱动，反对编造** | 所有skill都强调"先查数据再分析"，明确禁止用AI记忆/训练知识替代实时数据 |
| **多市场覆盖** | 除 `hk-quant-advisor` 专注港股外，其余均覆盖A股/港股/美股 |
| **免责声明** | 所有skill均声明不构成投资建议 |
| **腾讯证券数据生态** | `westock-*` 三个skill同源，数据来自腾讯自选股接口 |
| **技术指标支持** | 都涉及或可查询MACD/RSI/KDJ/均线等技术指标 |
| **估值维度** | 都关注PE/PB/ROE等基本估值指标 |

|------|-------------------|---------------|---------------|------------------|----------------|
| **定位** | 交易决策系统 | 数据查询 | 条件选股 | 投资顾问 | 自然语言选股 |

| **策略透明度** | ✅ 完全透明（每个条件可见） | N/A | ✅ 条件透明 | ⚠️ 框架透明但分析主观 | ❌ 黑盒（模型内部） |

### 3.3 思路层面的本质差异

这5个skill代表了**三种完全不同的投资辅助哲学**：

**哲学A：规则引擎派**（`hk-quant-advisor` + `westock-tool`）
- 把交易决策转化为可执行的布尔逻辑
- 优点：可回测、可审计、排除情绪干扰
- 缺点：规则设计本身仍依赖人的判断，参数敏感

**哲学B：数据基础设施派**（`westock-data`）
- 不做判断，只提供原材料
- 优点：纯粹、无偏见
- 缺点：本身不能独立使用，需要上层决策逻辑

**哲学C：框架展示派**（`westock-partner`）
- 把不同投资方法论具象化为人格，展示思维过程
- 优点：教育价值高，帮助用户建立自己的框架
- 缺点：主观成分大，最终仍需用户自己决策；高度依赖数据查询纪律

**哲学D：语义理解派**（`MX_StockPick`）
- 用大模型理解模糊意图，降低使用门槛
- 优点：灵活、易用、概念性查询能力强
- 缺点：可解释性差、精确性依赖模型理解、无规则可审计

## 四、作为严格量化投资者，如何选择？

### 4.1 什么是"严格量化投资"的核心要求

严格量化投资的本质是：**所有决策基于预先定义的规则，可回测，可审计，排除主观判断干扰**。具体要求：

1. **信号明确**：买入/卖出条件必须是可量化的布尔条件，不是"感觉强势"
2. **风控刚性**：止损、仓位必须事先确定并强制执行
3. **可回测**：历史数据上可以验证策略有效性
4. **数据可靠**：实时数据来源明确，不依赖模型记忆
5. **逻辑透明**：每笔交易都能追溯到具体触发条件

### 4.2 各 Skill 的量化投资适用性评分

| Skill | 信号明确性 | 风控刚性 | 可回测性 | 数据可靠性 | 逻辑透明度 | **综合评分** |
|-------|-----------|---------|---------|----------|----------|------------|
| `hk-quant-advisor` | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | **9.5/10** |
| `westock-data` | N/A | N/A | N/A | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | **数据层工具** |
| `westock-tool` | ⭐⭐⭐⭐ | ⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | **7/10** |
| `westock-partner` | ⭐⭐ | ⭐⭐ | ⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | **4/10** |
| `MX_StockPick` | ⭐⭐ | ⭐ | ⭐ | ⭐⭐⭐⭐ | ⭐⭐ | **3/10** |

### 4.3 分场景选择建议

#### 场景一：完整量化交易系统（港股）
**首选**：`hk-quant-advisor`（核心） + `westock-data`（数据支撑）

理由：
- `hk-quant-advisor` 是这5个skill里**唯一**满足严格量化投资全部要求的系统
- 信号生成有明确的量化条件（如"RSI 55~80 + 成交量比>2.0 + MACD零轴上方金叉"），全部条件必须满足
- 三层风控不可覆盖，止损规则硬性执行
- 有信号快照记录机制，支持事后复盘
- 有月度参数优化回路（虽然目前靠AI而非传统回测引擎）
- `westock-data` 作为数据源，提供实时行情/技术指标/财报等

**注意**：该系统目前尚不支持自动下单，所有指令仍需人工执行。但对于"半自动量化"场景（AI生成信号，人工确认执行）已经足够完善。

#### 场景二：构建选股池（全市场候选股筛选）
**首选**：`westock-tool` + `westock-data`

理由：
- `westock-tool` 的 `filter` 命令可以精确表达量化条件：`intersect([PE_TTM > 0, PE_TTM < 20, ROETTM > 15, MainNetFlow > 100000000])`
- 策略选股提供标准技术信号（MACD金叉等），可用于快速扫描信号股
- `westock-data` 做候选股的深度验证（K线/资金/财报）
- 两者结合可以实现：全市场扫描 → 缩小候选池 → 深度尽调 的标准量化选股流程

**不推荐用** `MX_StockPick`：虽然自然语言选股很方便，但"苹果产业链"这类描述对于严格量化来说太模糊，选出的结果无法用规则描述，不可回测验证。

#### 场景三：学习/验证投资框架
**参考使用**：`westock-partner`

理由：
- 六个高手的框架覆盖了量化投资的几乎所有主流方法论
- 文仔的PE Bands + EPS斜率估值框架可以直接转化为量化条件
- 洲仔的四层信号框架（政策/产业/资讯/资金）可以作为量化因子体系的设计参考
- 坤仔的支撑位买入逻辑可以转化为价格区间触发条件

**但要注意**：`westock-partner` 本身不是量化工具，它的输出是高手的"建议"而不是量化信号。不能直接用于量化交易，应该把它的方法论提炼成规则，再在 `westock-tool` 或 `hk-quant-advisor` 中实现。

### 4.4 严格量化投资者需要警惕的陷阱

**陷阱1：把 `westock-partner` 当量化系统用**

高手的建议再有逻辑，也是定性分析（"资金还没进来，别急"），不是量化信号。严格量化要求的是：资金净流入 > X万，且连续N日，才触发买入。两者之间有本质区别。

**陷阱2：只用 `westock-tool` 选股不考虑风控**

`westock-tool` 只做"选进来"，没有"何时卖出"、"止损在哪里"、"最大仓位多少"的回答。仅依赖选股工具没有风控体系，严格来说不算量化投资，更像是"量化选股 + 主观交易"。

**陷阱3：用 `MX_StockPick` 的自然语言替代量化条件**

"帮我找科技龙头"返回的列表，你不知道具体的筛选逻辑是什么，无法验证，无法回测，也无法用同样的标准在未来筛选。量化要求可复现性，自然语言查询不具备这个属性。

**陷阱4：忽视 `hk-quant-advisor` 的市场局限性**

该系统专注港股。如果目标市场是A股或美股，需要另行构建类似系统，或者将 `westock-tool` + `westock-data` 的组合升级为有完整风控逻辑的系统。

### 4.5 最终推荐路线图

```
严格量化投资者的工具链：

① 数据基础（必须）
   westock-data → 查询个股详情、K线、财报、技术指标

② 选股漏斗（必须）
   westock-tool → 条件选股（精确表达量化条件）
                  策略选股（标准技术信号扫描）

③ 交易决策系统（高度推荐，港股适用）
   hk-quant-advisor → 量化信号生成 + 三层风控 + 仓位管理

④ 方法论参考（选用）
   westock-partner → 框架学习，提炼成可量化的规则
   MX_StockPick   → 概念性选股探索，结果需用③②重新验证

⑤ 数据流向
   westock-data ←→ hk-quant-advisor （主数据源依赖关系）
   westock-tool + westock-data → 候选股初筛
   hk-quant-advisor → 量化信号 → 人工确认 → 执行
```

### 4.6 一句话总结

对于严格量化投资者：

- **`hk-quant-advisor`**：目前这5个skill中**唯一**真正意义上的量化系统，港股交易的核心工具；
- **`westock-data` + `westock-tool`**：量化选股的数据基础设施，必备组合；
- **`westock-partner`**：方法论教材，不是量化工具，参考价值高但不可直接用于量化决策；
- **`MX_StockPick`**：灵活探索工具，适合发现思路，但因为黑盒性质，不符合量化投资对可解释性的要求。

> 严格量化投资的本质是**纪律**，而不是技术。工具再好，如果不能严格执行规则（尤其是止损），都是徒劳。`hk-quant-advisor` 的价值不仅在于它的策略，更在于它把"不得不遵守的风控规则"内嵌到系统里，替人类克服了人性弱点。
